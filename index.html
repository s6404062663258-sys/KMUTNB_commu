Problem Statement ปัญหาที่พบ (Problem Statement)

Students lack an effective way to discover and join university activities that match their interests and skills. นักศึกษาขาดช่องทางที่มีประสิทธิภาพในการค้นหาและเข้าร่วมกิจกรรมภายในมหาวิทยาลัย ที่ตรงกับความสนใจและทักษะของตนเอง

Lack of Trust ขาดความน่าเชื่อถือ

Public social media groups do not verify users through university email accounts. The identity of posters and the reliability of shared information cannot always be confirmed. กลุ่มโซเชียลมีเดียสาธารณะไม่มีการยืนยันตัวตนด้วยอีเมลมหาวิทยาลัย ทำให้ไม่สามารถยืนยันตัวตนของผู้โพสต์ และไม่สามารถรับประกันความน่าเชื่อถือของข้อมูลได้เสมอไป

Unorganized Content ข้อมูลไม่เป็นระเบียบ

Different types of posts are mixed in a single feed without clear categories, making relevant information difficult to find. โพสต์ประเภทต่างๆ ถูกนำมารวมอยู่ในฟีดเดียวกันโดยไม่มีการแยกหมวดหมู่ที่ชัดเจน ทำให้ค้นหาข้อมูลที่ต้องการและเกี่ยวข้องได้ยาก

Difficulty Finding Partners หาเพื่อนร่วมกิจกรรมยาก

Posts seeking activity participants are mixed with general community posts, making activities difficult to discover and manage. โพสต์หาเพื่อนร่วมกิจกรรมปะปนกับโพสต์พูดคุยในคอมมูนิตี้ทั่วไป ทำให้โพสต์กิจกรรมถูกกลืนหาย ค้นพบยาก และจัดการได้ยาก

No Complete Workflow ขาดระบบจัดการกิจกรรมที่ครบวงจร

Existing social media groups lack a structured process for activity recommendations, join requests, approvals, membership management, and activity-specific communication. กลุ่มโซเชียลที่มีอยู่ไม่มีกระบวนการทำงานที่เป็นระบบ ตั้งแต่การแนะนำกิจกรรม ส่งคำขอเข้าร่วม อนุมัติสิทธิ์ จัดการสมาชิก ไปจนถึงการแชทพูดคุยเฉพาะกลุ่ม

Project Objectives วัตถุประสงค์ของโครงการ

Build a categorized community for verified students. สร้างคอมมูนิตี้แบบแยกหมวดหมู่ สำหรับนักศึกษาที่ยืนยันตัวตนแล้ว

Support the creation and management of university-related activities. รองรับการสร้างและจัดการกิจกรรมที่เกี่ยวข้องกับมหาวิทยาลัย

Recommend activities using explainable Hybrid Matching. แนะนำกิจกรรมด้วยระบบ Hybrid Matching ที่สามารถอธิบายเหตุผลได้

Connect requests, approvals, membership, and activity chat. เชื่อมต่อระบบคำขอเข้าร่วม การอนุมัติ สมาชิก และแชทกิจกรรมเข้าด้วยกัน

Support accountable anonymous posting. รองรับการตั้งโพสต์แบบไม่ระบุตัวตน ที่ตรวจสอบย้อนหลังได้เมื่อมีปัญหา

Provide reporting, admin review, and moderation records. มีระบบรายงานปัญหา การตรวจสอบโดยแอดมิน และบันทึกประวัติ (Audit Log)

The system has two main user roles: Verified Student and Administrator. A Verified Student can act as both an activity participant and an activity organizer. ระบบมีผู้ใช้งานหลัก 2 บทบาท: นักศึกษาที่ยืนยันตัวตนแล้ว และ ผู้ดูแลระบบ โดยนักศึกษาสามารถเป็นได้ทั้งผู้เข้าร่วมและผู้สร้างกิจกรรม

01 PRIMARY USER

Verified Students นักศึกษาที่ยืนยันตัวตนแล้ว

Community Userผู้ใช้งานคอมมูนิตี้

Creates profiles, browses categorized posts, creates normal or anonymous posts, comments, reports inappropriate content, and tracks report status. สร้างโปรไฟล์ เลือกดูโพสต์ตามหมวดหมู่ สร้างโพสต์ปกติหรือแบบไม่ระบุตัวตน แสดงความคิดเห็น รายงานเนื้อหาที่ไม่เหมาะสม และติดตามสถานะการรายงาน

Activity Participantผู้เข้าร่วมกิจกรรม

Discovers activities, receives rule-based recommendations, views match reasons, requests to join, tracks request status, accesses activity chat after approval, and submits feedback. ค้นหากิจกรรม รับการแนะนำกิจกรรมที่ตรงกับตนเอง ดูเหตุผลการจับคู่ ส่งคำขอเข้าร่วม ติดตามสถานะคำขอ เข้าถึงแชทกิจกรรมเมื่อได้รับอนุมัติ และประเมินผล

Activity Ownerผู้สร้างกิจกรรม

Creates and manages activities, defines required interests and skills, reviews join requests, manages members, communicates through activity chat, completes activities, and reviews feedback. สร้างและจัดการกิจกรรม กำหนดทักษะและความสนใจที่ต้องการ ตรวจสอบคำขอเข้าร่วม จัดการสมาชิก สื่อสารผ่านแชทกิจกรรม กดจบกิจกรรม และดูผลประเมิน

02 ADMIN

Administratorผู้ดูแลระบบ

An authorized user responsible for maintaining platform safety and integrity. ผู้ใช้งานที่ได้รับสิทธิ์พิเศษ รับผิดชอบในการดูแลความปลอดภัยและความเรียบร้อยของแพลตฟอร์ม

  • Reviewing user reportsตรวจสอบรายงานจากผู้ใช้
  • Moderating contentจัดการและควบคุมเนื้อหา
  • Managing report statusอัปเดตสถานะการรายงานปัญหา
  • Investigating anonymous postsตรวจสอบโพสต์นิรนามเมื่อทำผิดกฎ
  • Handling account restrictionsจัดการสถานะและระงับบัญชีผู้ใช้
  • Maintaining immutable audit logsดูแลบันทึกประวัติการทำงาน (Audit Log)

Scoring Algorithm Exampleตัวอย่างการคิดคะแนน (Scoring Algorithm)

User Profile Bio:ประวัติในโปรไฟล์ผู้ใช้:

"Interested in building websites and designing user experiences." "สนใจทำหน้าเว็บและออกแบบประสบการณ์ผู้ใช้"

Activity Requirement:ความต้องการของกิจกรรม:

"Looking for peers to help with UI/UX for a Web Application." "หาเพื่อนช่วยทำ UI/UX สำหรับ Web Application"

Rule Score = Skill 40% + Interest 30% + Major 15% + Year 15%
Semantic Score = Cosine similarity (Profile Vector, Activity Vector) × 100
Hybrid Score = (Rule Score × 0.80) + (Semantic Score × 0.20)

Tools & Technology Matrixเครื่องมือและเทคโนโลยี (Tools & Technology)

Architecture Layerเลเยอร์สถาปัตยกรรม Technology Stackเทคโนโลยี Core Responsibilityความรับผิดชอบหลัก
User Interfaceส่วนแสดงผล (UI) React, Tailwind CSS, shadcn/ui Mobile-first UI, Components, Feedback StatesUI รองรับมือถือเป็นหลัก, คอมโพเนนต์, แสดงสถานะตอบสนอง
Applicationแอปพลิเคชัน Next.js, TypeScript Routing, Server Actions, Business Logic, Type Safetyการจัดการเร้าติ้ง, Server Actions, ลอจิกธุรกิจ, ตรวจสอบชนิดตัวแปร
Input Safetyความปลอดภัยข้อมูลเข้า React Hook Form, Zod Form State Management and Runtime Validationจัดการสถานะฟอร์ม และตรวจสอบข้อมูลระหว่างทำงาน
Identity & Dataการยืนยันตัวตน & ข้อมูล Google OAuth, Supabase Auth, PostgreSQL Accounts, Sessions, Schemas, Queries, Constraints, Transactionsบัญชีผู้ใช้, เซสชัน, โครงสร้างข้อมูล, คำสั่งสืบค้น, ข้อจำกัด, ทรานแซกชัน
Authorizationการอนุญาตสิทธิ์ Server RBAC, Row Level Security (RLS) Role, Owner, Member, and Row-level Permissionsสิทธิ์ตามบทบาท, เจ้าของ, สมาชิก และการป้องกันระดับแถวข้อมูล
Matching Coreแกนระบบจับคู่ Rule-Based Matching Engine Skill, Interest, Major, and Year Scoringระบบให้คะแนนตามทักษะ ความสนใจ สาขา และชั้นปี
AI Matchingการจับคู่ด้วย AI Gemini Text Embedding, pgvector Semantic Vector Search and Cosine Similarityค้นหาด้วยความหมายเชิงเวกเตอร์ และหาค่า Cosine Similarity
Hybrid Rankingการจัดอันดับแบบผสม Rule Score + Semantic Score Final Match Score Calculation and Activity Rankingคำนวณคะแนนจับคู่รวมขั้นสุดท้าย และจัดอันดับกิจกรรม
Realtime Supabase Realtime Instant Event Broadcasting (UI updates, not authorization)ถ่ายทอดเหตุการณ์แบบทันที (เพื่ออัปเดต UI ไม่เกี่ยวกับเรื่องสิทธิ์)
Admin Next.js, PostgreSQL, Audit Log Report Review, Moderation System, User Managementระบบตรวจสอบรายงาน จัดการเนื้อหา และจัดการผู้ใช้งาน
